# Paper Replication Map (Config + Notebooks)

This document maps each paper result block to:

1. the exact simulation config file(s),
2. preprocessing steps (when scenario-combination files must be regenerated),
3. postprocessing notebooks.

## Execution conventions

Run from repository root:

```bash
python -m project.main -c <config_path>
```

Batch run all JSON configs in a folder (recursive):

```bash
python -m project.runs -d <config_folder>
```

Outputs are written to `project/output/<timestamp>/`.

## 1) Market failures

### 1.1 One-by-one market failures + "all together"

- Config: `project/config/market_failures/market_failures_bundle.json`
- Policy-scenario source: `project/input/policies/policies_mf.json`
- Scenarios generated: `Reference` + `credit_constraint`, `ghg_externality`, `landlord_dilemma`, `multi_family`, `present_bias`, `statusquo_bias`, `health_cost`, `all`
